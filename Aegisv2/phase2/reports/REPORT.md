## Executive Summary

**Executive Summary**

A comprehensive penetration test was conducted against the target environment at https://appeasing-craftily-scorer.ngrok-free.dev, comprising 15 distinct test cases designed to evaluate the security posture of the application. The overall security posture is assessed as **moderate**, with the majority of tested controls performing as expected. However, the identification of a single high-severity vulnerability indicates a significant gap in access control mechanisms that requires immediate remediation. No critical vulnerabilities were discovered, which suggests that foundational security measures are largely in place, but the presence of a high-risk finding underscores the need for targeted improvements.

The most critical finding is a **Broken Access Control vulnerability** in the form of an Insecure Direct Object Reference (IDOR) at the endpoint `/api/files`. This flaw allows an authenticated user to access files belonging to other users by manipulating file identifiers in API requests. The impact is substantial, as it could lead to unauthorized disclosure of sensitive data, including personally identifiable information (PII), proprietary documents, or other confidential materials. The vulnerability was confirmed through manual testing and demonstrates a clear bypass of intended authorization checks.

**Immediate actions required** include implementing robust server-side authorization checks for all API endpoints, particularly `/api/files`, to verify that the requesting user has explicit permission to access the requested resource. This should be achieved by validating user identity against resource ownership or role-based access controls (RBAC) before serving any file. Additionally, a thorough review of all other endpoints for similar IDOR patterns is recommended, along with the adoption of randomized, non-sequential identifiers (e.g., UUIDs) to reduce the predictability of resource references. A follow-up retest should be scheduled after remediation to confirm the effectiveness of these measures.

## Detailed Findings


### 1. IDOR - Access Other User's Files

| Property | Value |
|----------|-------|
| **Category** | Broken Access Control |
| **Severity** | ![High](https://img.shields.io/badge/High-orange) |
| **Endpoint** | `/api/files` |
| **OWASP** | A01:2021-Broken Access Control |
| **CWE** | CWE-639 |

**Payload:**
```
{"fileId": 1234}
```

**Evidence:**
The endpoint returned a 200 status code and file content for fileId 1234 without requiring authorization, indicating a horizontal privilege escalation vulnerability.

**Recommendation:**
Implement proper access control checks to ensure that users can only access files they are authorized to view. Use session-based or token-based authentication and validate ownership or permissions before returning file data.
